# Testing

SafeRE has an extensive test suite organized into several layers: unit
tests for internal components, integration tests using RE2's reference
test data, cross-engine tests comparing against `java.util.regex`,
regression tests for specific bugs, and functional tests for the public
API.  Many of these tests are parameterized — for example, the exhaustive
test alone runs over 10 million individual assertions parsed from test
data originally generated by RE2 C++ and obtained via RE2/J.

## Test Structure

Most tests are parameterized: each (pattern, text, expected) tuple from
the test data files becomes its own JUnit test case.

### Integration Tests

These validate SafeRE against reference test data from RE2 and RE2/J.
Parameterized tests make up the bulk of the suite:

| Test File | What It Tests |
|---|---|
| `RE2SearchTest` | Test vectors parsed from `re2-search.txt` |
| `DfaNfaTest` | DFA/NFA agreement across a systematic pattern × input matrix |
| `RE2PosixTest` | AT&T POSIX regex test data (`basic.dat`, `nullsubexpr.dat`, `repetition.dat`) |
| `RE2FindTest` | Find/match tests from RE2/J |
| `SimplifierTest` | Simplification tests from RE2 C++ `simplify_test.cc` |
| `RE2ReplaceTest` | Replace tests from RE2/J |
| `RE2RegressionTest` | Bug regressions and edge cases from RE2 C++ `re2_test.cc` |
| `RE2ExhaustiveTest` | Parses `re2-exhaustive.txt.gz` (from RE2/J; ~10 million assertions in a single test) |

### Cross-Engine Tests

These compare SafeRE's results against `java.util.regex` to verify
JDK-compatible behavior:

| Test File | What It Tests |
|---|---|
| `CrossEngineTest` | Parameterized comparison of SafeRE vs `java.util.regex` across diverse patterns |
| `CrossEngineExhaustiveTest` | Exhaustive cross-engine comparison: multiline anchors, case folding, word boundaries, character classes, line endings, and more (generates millions of assertions) |
| `RandomTest` | Random fuzz testing against `java.util.regex` with fixed seeds |

### Unit Tests

These test SafeRE's internal classes directly:

| Test File | What It Tests |
|---|---|
| `ParserTest` | Stack-based operator-precedence regex parser |
| `CompilerTest` | Thompson NFA construction |
| `NfaTest` | Pike VM execution engine |
| `DfaTest` | Lazy DFA engine |
| `BitStateTest` | BitState backtracking engine |
| `OnePassTest` | OnePass automaton detection and matching |
| `RegexpTest` | Regexp AST node operations |
| `CharClassTest` | Character class building and matching |
| `UnicodeTablesTest` | Unicode property tables, predefined/property classes, case folding |
| `UtilsTest` | Utility functions |
| `InstTest` | Instruction structure and initialization |
| `WalkerTest` | Iterative Regexp tree traversal |
| `ProgTest` | Compiled instruction programs |
| `ParseFlagsTest` | Parse flag constants and combinations |
| `RegexpOpTest` | Regexp operator enum |
| `EmptyOpTest` | Empty assertion operators |
| `InstOpTest` | Instruction opcode enum |

### Functional Tests

These test the public API as users see it:

| Test File | What It Tests |
|---|---|
| `MatcherTest` | `matches()`, `find()`, `group()`, replacement, regions, Unicode, state management |
| `PatternTest` | `Pattern.compile()`, group counting, `quote()`, named groups, unsupported features |
| `PatternSetTest` | Multi-pattern matching builder, anchor modes, and execution |

### Regression Tests

These capture specific bugs found during development and validation:

| Test File | What It Tests |
|---|---|
| `AnchorOptTest` | Anchor optimization and one-pass matching for anchored patterns |
| `Bug3Test` | Multiline mode behavior with trailing newlines |
| `Issue42RegressionTest` | Leftmost match selection with `$\|\b` alternation; multiline `^\|$` matching |
| `Issue55RegressionTest` | Zero-width match handling in repetition (`*`, `+`, nullable bodies, `\B\|a`) |

### Meta Tests

| Test File | What It Tests |
|---|---|
| `LicenseHeaderTest` | Verifies every Java source file has the BSD 3-Clause license header |

## Tests Ported from RE2 and RE2/J

A significant portion of SafeRE's integration test suite is ported from
the RE2 family of libraries.  This was an intentional strategy: by reusing
established test data and test cases, we can be confident that SafeRE produces
the same results as the reference implementations.

### Test Data Files

The test data files in `safere/src/test/resources/` were taken from
[RE2/J](https://github.com/google/re2j)'s
[`testdata/`](https://github.com/google/re2j/tree/master/testdata)
directory.  RE2/J in turn obtained them from the
[RE2](https://github.com/google/re2) C++ project, where they were
generated by RE2's exhaustive test infrastructure.

| File | Origin | Description |
|---|---|---|
| `re2-exhaustive.txt.gz` | RE2 C++ (via RE2/J) | ~10 million systematically generated (pattern, text, expected) tuples covering all operator combinations |
| `re2-search.txt` | RE2 C++ (via RE2/J) | Search test vectors with full-match, partial-match, and submatch results |
| `basic.dat` | AT&T POSIX (via RE2/J) | AT&T POSIX basic regex test data |
| `nullsubexpr.dat` | AT&T POSIX (via RE2/J) | AT&T POSIX null subexpression tests |
| `repetition.dat` | AT&T POSIX (via RE2/J) | AT&T POSIX repetition tests |

### Test Code Ported from RE2/J

[RE2/J](https://github.com/google/re2j) is a Java port of RE2.
The following test code was ported from RE2/J and adapted to SafeRE's API
and test framework (JUnit 6 + AssertJ):

- **`RE2FindTest`** — Find/match test data from RE2/J's `FindTest.FIND_TESTS`.
  Adapted to compare matched text instead of byte offsets, since SafeRE
  operates on Java chars (UTF-16) rather than UTF-8 bytes.
- **`RE2ReplaceTest`** — Replace test cases from RE2/J's `RE2ReplaceTest`.
- **`RE2ExhaustiveTest`** — Test runner ported from RE2/J's `ExecTest`
  (itself ported from Go's `regexp/syntax/exec_test.go`).  Parses
  `re2-exhaustive.txt.gz` and runs each test case.
- **`RE2SearchTest`** — Test runner ported from RE2/J's `ExecTest`.
  Parses `re2-search.txt`.
- **`RE2PosixTest`** — Test runner ported from RE2/J's `ExecTest`.
  Parses the AT&T `.dat` files.
- **`PatternTest`** (subsections) — Compile validation, group counting, and
  `quote()` round-trip tests from RE2/J's `RE2CompileTest`,
  `RE2TestNumSubexps`, and `RE2QuoteMetaTest`.
- **`UnicodeTablesTest`** (subsection) — `simpleFold()` tests from RE2/J's
  `UnicodeTest`.

### Test Code Ported from RE2 C++

The following test code was ported from the C++ RE2 test suite and
rewritten in Java:

- **`RE2RegressionTest`** — Bug regressions and edge cases from
  `re2_test.cc`.
- **`SimplifierTest`** — Simplification tests from `simplify_test.cc`.
- **`RandomTest`** — Random/fuzz testing approach from `random_test.cc`,
  adapted to compare against `java.util.regex` instead of PCRE.
- **`PatternSetTest`** (subsection) — Set tests from `set_test.cc`.

## Running Tests

```bash
# Run all tests
mvn test -pl safere

# Run a specific test class
mvn test -pl safere -Dtest=ParserTest

# Run tests with coverage report (JaCoCo)
mvn verify -pl safere -Pcoverage

# Run crosscheck module tests
mvn test -pl safere-crosscheck -am

# Run SafeRE tests and generated public API tests through the crosscheck facade
mvn verify -pl safere,safere-crosscheck -am -Pcrosscheck-public-api-tests

# Run generated public API tests only, after safere has already been built locally
mvn verify -pl safere-crosscheck -Pcrosscheck-public-api-tests

# Run the long generated character-class syntax sweep
tools/exhaustive/run-character-class-sweep.sh --threads=8 \
  --output-dir=target/exhaustive-reports/character-class-sweep-full

# Run a bounded generated-case range for local debugging
tools/exhaustive/run-character-class-sweep.sh --threads=4 --range=:1000000 \
  --output-dir=target/exhaustive-reports/character-class-sweep-smoke

# Run Jazzer fuzz targets in regression mode
mvn test -pl safere-fuzz
```

Coverage reports are generated at
`safere/target/site/jacoco/index.html`.  Note that JaCoCo is disabled
by default; the `-Pcoverage` profile is required to enable it.

The generated character-class syntax sweep is intentionally outside the
ordinary JUnit path. It compares SafeRE with `java.util.regex.Pattern` over
large deterministic character-class expression spaces, including intersection,
ampersand, zero-width quoted syntax, comments-mode trivia, nested class,
property, and range-tail shapes. Use `--range=start:end` to run a contiguous
generated-case slice, and omit either bound to start at 0 or run to the end.
The sweep writes every divergence to JSONL as it runs so interrupted runs still
leave useful repro data.

## Differential Testing with safere-crosscheck

The [safere-crosscheck](../safere-crosscheck/) module provides a
differential-testing facade that runs both SafeRE and `java.util.regex` on
every operation, comparing results and throwing `CrosscheckException` if they
diverge. This catches behavioral differences that aren't directly covered by
unit tests.

To use it, change your import from `org.safere.Pattern` to
`org.safere.crosscheck.Pattern` (and likewise for `Matcher`). Every API
call is then executed on both engines and compared automatically.

The facade records a trace of all API calls. When a divergence is detected,
the exception includes the full trace, providing enough context to reproduce
and report the bug. See
[safere-crosscheck/README.md](../safere-crosscheck/README.md) for details.

For pre-PR validation, use the combined reactor command above. It runs the
`safere` tests once and then runs generated copies of SafeRE public API tests
through the crosscheck facade. The `-am` flag ensures Maven rebuilds the local
`safere` module instead of resolving a stale installed artifact.

## Fuzz Testing with Jazzer

The [safere-fuzz](../safere-fuzz/) module contains Jazzer fuzz targets that
use `safere-crosscheck` as their oracle. By default, they run in regression
mode over checked-in seed inputs:

```bash
mvn -pl safere-fuzz test
```

For coverage-guided fuzzing, set `JAZZER_FUZZ=1` and select a single target:

```bash
JAZZER_FUZZ=1 mvn -pl safere-fuzz -Dtest=MatchFuzzer test
```

When a fuzzer finds a valid SafeRE/JDK divergence or crash, minimize the input
and add a normal JUnit regression test before fixing the bug. See
[safere-fuzz/README.md](../safere-fuzz/README.md) for target descriptions and
workflow details.
